# HyPaper

HyPaper is an open-source paper trading backend for [HyperLiquid](https://hyperliquid.xyz) perpetual futures and [Limitless](https://limitless.exchange) prediction markets. It mirrors the real exchange APIs — same request shapes, same response shapes, same WebSocket protocol — but uses simulated money instead of real funds.

Swap `api.hyperliquid.xyz` for your HyPaper URL and your existing HL bot just works. The Limitless integration adds parallel `/lm/*` API routes for binary outcome market trading.

## How it works

```
HL WebSocket Feed ──┐
                    ├──> Worker Process ──> Redis (all hot state)
Limitless SDK ──────┘     - HL price updates       |
                          - HL order matching       | reads
                          - LM price updates        v
                          - LM order matching   Hono API Server
                          - LM resolution       POST /exchange      (HL)
                                                POST /info          (HL)
                                                POST /hypaper       (HL admin)
                                                POST /lm/exchange   (Limitless)
                                                POST /lm/info       (Limitless)
                                                POST /lm/hypaper    (Limitless admin)
                                                     |
                         EventBus <──────────────────+
                           |
                           v
                      WebSocket Server (/ws)
                      - allMids, l2Book
                      - orderUpdates, userFills
                      - lmOrderUpdates, lmUserFills
```

1. **Worker** streams live market data from HyperLiquid and Limitless, filling paper orders on every price tick
2. **Redis** holds all hot state: prices, positions, orders, fills, balances (separate namespaces per exchange)
3. **API** mirrors HL's endpoints (zero code changes for HL bots) + custom `/lm/*` routes for Limitless
4. **WebSocket** pushes real-time updates to connected clients

## Tech stack

| Component | Technology |
|-----------|-----------|
| Runtime | Node.js + TypeScript |
| HTTP | [Hono](https://hono.dev) |
| WebSocket | [ws](https://github.com/websockets/ws) |
| State | Redis via ioredis |
| Persistence | PostgreSQL via Drizzle ORM |
| Math | [decimal.js](https://github.com/MikeMcl/decimal.js) (zero floating-point errors) |
| Limitless SDK | [@limitless-exchange/sdk](https://www.npmjs.com/package/@limitless-exchange/sdk) |
| Validation | Zod |
